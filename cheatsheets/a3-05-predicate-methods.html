5 · Predicate methods

Yes/no questions about strings. All return True or False. Check the whole string — one bad char ruins it.
Loop Week · Day 3 · Block A.3 · Subtopic 5

Character-type checks

"12345".isdigit()           →  True       (every char is 0-9)
"abc".isalpha()             →  True       (every char is a letter)
"abc123".isalnum()          →  True       (letters and/or digits)
"   ".isspace()             →  True       (every char is whitespace)
"Hello".isupper()           →  False      (mixed case)
"HELLO".isupper()           →  True
"hello".islower()           →  True

# Mixed = False
"abc 123".isalnum()         →  False      (space breaks it)
"abc.com".isalpha()         →  False      (dot breaks it)

Prefix / suffix checks

"hello.py".startswith("hello")        →  True
"hello.py".startswith("hi")           →  False
"hello.py".endswith(".py")            →  True

# Pass a tuple to check multiple options at once
"hello.py".endswith((".py", ".js", ".txt"))    →  True
"https://...".startswith(("http://", "https://"))    →  True
Tuple shortcut · pass a tuple like (".py", ".js") to mean "any of these." Cleaner than three or conditions.

The empty-string rule

"".isalpha()       →  False
"".isdigit()       →  False
"".isspace()       →  False
"".isalnum()       →  False
"".isupper()       →  False
"".islower()       →  False
Empty strings fail every predicate. They need at least one character to say "yes." This bites when you forget to check for empty input first.

The other predicates (occasionally useful)

"Hello World".istitle()        →  True       (every word starts with caps)
"Hello world".istitle()        →  False      (second word not capitalized)

"123".isnumeric()              →  True       (broader than isdigit — fractions, Roman numerals too)
"3.14".isdigit()               →  False      (the dot breaks it!)
"3.14".isnumeric()             →  False      (same problem)

"my_var".isidentifier()        →  True       (valid Python variable name)
"123abc".isidentifier()        →  False      (can't start with digit)
Trap · "3.14".isdigit() is False. The decimal point breaks it. Don't use .isdigit() to check for "is this a number" — use try/except float(...) instead.

Real-world patterns

# 1. File extension check (use tuple)
if filename.endswith((".jpg", ".png", ".gif")):
    process_image(filename)

# 2. URL routing
if path.startswith("/api/"):
    handle_api()
elif path.startswith("/static/"):
    serve_static()

# 3. Validate input is digits only (then convert)
if user_input.isdigit():
    age = int(user_input)
else:
    print("Please enter a number")

# 4. Count digits in a mixed string
text = "abc1def2ghi3"
digit_count = sum(1 for c in text if c.isdigit())     →  3

Lock these

  1. All predicates return True/False and check the whole string. One bad char = False.
  2. Empty string is always False for every predicate.
  3. The big four: isdigit, isalpha, isalnum, isspace.
  4. startswith / endswith accept a tuple for "any of these."
  5. .isdigit() does NOT handle decimals or signs. For real "is this a number," use try: float(s).
