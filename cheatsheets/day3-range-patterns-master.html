0 · The mental model

Every range answers ONE question: "where in the sequence do I walk, and how?"

The shape of the range is determined by what you're DOING in the loop. Three questions to ask:

  1. Do I need to peek at neighbors? → trim the range (skip the first or last position)
  2. Do I need a chunk of N consecutive items? → window math (stop early enough that the chunk fits)
  3. Does the inner range depend on the outer? → shape math (triangle, pyramid, etc.)
Master rule · range(N) walks 0, 1, 2, ..., N-1. It stops before N. To include the last position, the argument to range must be (last position + 1).

1 · Walk every position

No special needs — just visit each item once.
range(len(s)) · enumerate(s) · for x in s
# Indexed walk
for i in range(len(s)):
    use(s[i])

# Indexed + value (Pythonic — use this 80% of the time)
for i, ch in enumerate(s):
    use(i, ch)

# Just values (no index needed)
for ch in s:
    use(ch)

Why this range?

You want every index from 0 to len(s) - 1. range(len(s)) gives exactly that.

When to use which · for x in s if you don't need the index. enumerate if you need both. range(len(s)) when you need the index for math (jumping around, comparing positions).

2 · Need to peek at neighbors

Trim the range so the neighbor lookup stays inside the sequence.
NeedRangeWhy
Look at s[i+1] (right neighbor)range(len(s) - 1)Stop 1 early — at last position, there is no s[i+1]
Look at s[i-1] (left neighbor)range(1, len(s))Start at 1 — at position 0, there is no s[i-1]
Look at BOTH neighborsrange(1, len(s) - 1)Trim both edges
 
# Right neighbor — duplicate-detection example
for i in range(len(s) - 1):
    if s[i] == s[i+1]:
        print(f"duplicate at {i}")

# Left neighbor — comparing each to the previous
for i in range(1, len(s)):
    if s[i] > s[i-1]:
        print(f"increasing at {i}")

The dominos lock

"Need a right neighbor? Stop 1 early." This is the rule. The "-1" is because the last position has no neighbor to its right — you'd crash with IndexError trying to read past the end.

Trap · forgetting the -1 → IndexError on the last iteration when you try s[i+1].

3 · RLE-style — compare each item to the PREVIOUS one

Walk starting from position 1, look back at i-1.
range(1, len(s))
# Run-length encoding compress (Pattern 4)
count = 1
for i in range(1, len(s)):
    if s[i] == s[i-1]:
        count += 1
    else:
        # flush previous group
        count = 1

Why start at 1 (not 0)?

Because at each step we look BACK at s[i-1]. If we started at i=0, s[i-1] would be s[-1] — which Python treats as the LAST character (wraparound). That would be a bug. Starting at i=1 means s[i-1] is always a real earlier position.

Initialization quirk · with RLE, you initialize count = 1 BEFORE the loop because you've already "seen" s[0]. The loop starts comparing from position 1.

4 · Sliding window — chunks of N consecutive items

Stop early enough that an N-item window still fits.
range(len(s) - window_size + 1)
# Substring scanning (Pattern 5)
for i in range(len(s) - window_size + 1):
    window = s[i:i+window_size]   # window of `window_size` items
    if is_unique(window):
        return True

Why this formula?

  • - window_size → leaves room for the window to fit at the right edge
  • + 1 → to include the last valid starting position (counting from 0 needs the +1)

For s = "hello" (length 5) with window size 3, valid starts are positions 0, 1, 2 → 3 positions. Formula: 5 - 3 + 1 = 3. ✓

Generalized dominos · the dominos rule from §2 is a special case of this — when window_size = 2 (you + 1 neighbor), the formula becomes range(len - 2 + 1) = range(len - 1).
Trap · using just range(len(s)) instead → at the last positions, your slice s[i:i+window_size] would be SHORT (cut off by the string end). Silent bug — the slice still works, it's just incomplete.

5 · Two-pointer from opposite ends (walking inward)

Not a range() — manual pointers, while loop, stops when they meet.
left, right = 0, len(s) - 1
while left < right: ...
# Palindrome (Pattern 3)
def is_palindrome(s):
    left, right = 0, len(s) - 1
    while left < right:
        if s[left] != s[right]:
            return False
        left += 1
        right -= 1
    return True

# Reverse via swap (build a list, swap, join)
def reverse_string(s):
    chars = list(s)
    left, right = 0, len(chars) - 1
    while left < right:
        chars[left], chars[right] = chars[right], chars[left]
        left += 1
        right -= 1
    return "".join(chars)

Why the stop condition left < right?

  • Pointers walk inward from opposite ends. When they meet (or cross), we're done.
  • For an odd-length string, they meet at the middle char — which doesn't need to compare with itself.
  • For an even-length string, they cross over without meeting — last pair is the two innermost chars.
When to use this · palindrome checks, in-place reverse, pair-based comparisons (sum to target with sorted array, etc.). Lets you walk half the sequence — twice as fast as the slice-and-compare approach for long strings with early failures.

6 · Two-pointer same direction (window expand/contract)

Both pointers move right. One scouts ahead (right), the other lags behind (left). The gap is your window.
left = 0
for right in range(len(s)):
    # expand or contract window
# Longest substring without repeating chars (LC 3 — Block B.3)
def longest_unique_substring(s):
    seen = set()
    left = 0
    best = 0
    for right in range(len(s)):
        # Shrink window from the left until s[right] is unique
        while s[right] in seen:
            seen.remove(s[left])
            left += 1
        seen.add(s[right])
        best = max(best, right - left + 1)
    return best

# Also used for: max sum subarray of size K, smallest window
# containing all chars of X, etc.

Why this differs from §5

§5 has pointers moving in OPPOSITE directions (one right, one left, meeting in the middle). §6 has BOTH pointers moving in the SAME direction (both right), with the WINDOW between them growing or shrinking.

When to use this · variable-size window problems where you can't know the window size up-front. Real sliding window (the proper version of Pattern 5). Comes up in interview problems all the time — Day 6+ territory.

7 · Variable-width inner ranges (shapes)

Inner range depends on the outer loop variable. Used for triangles, pyramids, diamonds.

Key formulas

FormulaWhat it producesRow by row
range(i + 1)Growing count, row 0 has 11, 2, 3, 4, 5
range(i)Same shape, row 0 has 0 (empty)0, 1, 2, 3, 4
range(height - i - 1)Leading spaces (shrinking)4, 3, 2, 1, 0
range(2 * i + 1)Pyramid stars (odd, growing)1, 3, 5, 7, 9
range(2 * (height - i) - 1)Inverted pyramid stars (odd, shrinking)9, 7, 5, 3, 1
range(i) for inverted spacesLeading spaces (growing)0, 1, 2, 3, 4
range(1, i + 2)Numbers 1 to i+1[1], [1,2], [1,2,3]

The mental model

Imagine a center column at height - 1. Each row extends i steps to the LEFT and i steps to the RIGHT of that center, plus the center itself.

  • Total stars per row = i + 1 + i = 2i + 1 (left + center + right)
  • Leading spaces = height - 1 - i (to push leftmost star to its column)
  • For inverted: count rows REMAINING (height - i) instead of rows DONE
# Left-aligned triangle (Block C.2)
for i in range(height):
    for j in range(i + 1):
        print("*", end="")
    print()

# Right-aligned triangle (Block C.4) — two inner loops per row
for i in range(height):
    for s in range(height - i - 1):       # spaces shrink
        print(" ", end="")
    for j in range(i + 1):                 # stars grow
        print("*", end="")
    print()

# Centered pyramid (Block D.1) — odd stars, centered
for i in range(height):
    for s in range(height - i - 1):       # leading spaces
        print(" ", end="")
    for j in range(2 * i + 1):             # odd-numbered stars
        print("*", end="")
    print()
Decoupling insight · the spaces loop and the stars loop are INDEPENDENT computations. SWAP their order to change the alignment (spaces-first → right-aligned; stars-first → left-aligned with invisible trailing spaces).

8 · Walk backwards (descending range)

When you need to iterate from high to low.
range(start, stop, -1) · step is NEGATIVE
# Walk indices from len(s) - 1 down to 0
for i in range(len(s) - 1, -1, -1):
    use(s[i])

# Diamond bottom half (Block D.3 Approach A)
# After top pyramid, walk back DOWN to make the mirror
for i in range(height - 2, -1, -1):       # height-2 down to 0
    for s in range(height - i - 1):
        print(" ", end="")
    for j in range(2 * i + 1):
        print("*", end="")
    print()

Why this format?

  • Start = the FIRST value you want (e.g., len(s) - 1 for last position)
  • Stop = one PAST where you want to stop (e.g., -1 to include 0)
  • Step = -1 to count down
The "stop = -1" trick · range stops BEFORE the stop value. To INCLUDE 0 going backwards, you have to stop at -1 (the position before 0).
Alternative for simple reverse · reversed(s) or s[::-1] are usually cleaner if you just want the values backwards. range(..., -1, -1) is for when you need the INDICES going backwards.

9 · Step / skip patterns

Take every Nth item instead of every item.
range(start, stop, step)
range(0, 10, 2)     # 0, 2, 4, 6, 8         (every 2nd)
range(0, 10, 3)     # 0, 3, 6, 9            (every 3rd)
range(1, 10, 2)     # 1, 3, 5, 7, 9         (odd numbers)

# Walk a string every other char
for i in range(0, len(s), 2):
    print(s[i])

# Slice equivalent
s[::2]              # same chars as above, simpler if you just want them
When to use range vs slice · range(0, N, step) when you need the index for other math. s[::step] when you just want the values.

10 · Decision tree — pick a range by what you need

  1. Do you need the index at all?
    No → for x in seq. Done.
    Yes → continue.
  2. Do you need the index AND the value?
    Yes → for i, x in enumerate(seq). Done.
    Just the index → continue.
  3. Do you need to peek at a neighbor?
    Right neighbor (s[i+1]) → range(len(s) - 1).
    Left neighbor (s[i-1]) → range(1, len(s)).
    Both neighbors → range(1, len(s) - 1).
  4. Do you need a chunk of N items?
    Fixed-size sliding window → range(len(s) - N + 1).
  5. Do you need to compare from both ends?
    Palindrome / reverse / pair → left, right = 0, len(s) - 1; while left < right.
  6. Do you need a variable-size window?
    Longest unique substring / smallest containing X → two pointers same direction (§6).
  7. Is the inner range a function of the outer loop variable?
    You're building a shape. Use formulas from §7 (range(i + 1), range(2*i + 1), etc.).
  8. Walking backwards?
    Indices descending → range(len(s) - 1, -1, -1). Or use reversed() / s[::-1] if you just want values.
  9. Skipping items?
    Every Nth → range(0, len(s), N).

The 5 formulas to actually memorize

If you remember these, you can derive the rest.
FormulaMeaning
range(len(s))Walk every position
range(len(s) - 1)Stop 1 early — for right-neighbor access
range(1, len(s))Start at 1 — for left-neighbor access (and RLE-style)
range(len(s) - window + 1)Sliding window valid starts
range(i + 1)Inner depends on outer — triangle shape

Everything else (2*i + 1, height - i - 1, range(start, stop, -1)) is built from the same logic — counting from 0, leaving room, including endpoints. Once these 5 click, the rest is variations on the theme.
