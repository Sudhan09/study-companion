7 · f-string format specs

The mini-language after the colon inside {}. Controls width, alignment, fill, precision, type.
Loop Week · Day 3 · Block A.3 · Subtopic 7

The recipe — read it left to right

{ value : [fill] [align] [width] [.precision] [type] }

Most specs use 2-3 of these slots. All of them are optional after the colon.

Alignment — the 3 symbols

name = "Alice"

f"{name:10}"        →  "Alice     "       (width 10, default LEFT for strings)
f"{name:<10}"       →  "Alice     "       (explicit left)
f"{name:>10}"       →  "     Alice"       (right-align)
f"{name:^10}"       →  "  Alice   "        (center)

# For numbers, default alignment is RIGHT
f"{42:6}"           →  "    42"            (right-aligned without symbol)
Symbols · < left, > right, ^ center. Default differs by type: strings go left, numbers go right.

Fill character — goes BEFORE the align symbol

x = 42
name = "Bob"

f"{x:0>5}"          →  "00042"            (pad left with 0s)
f"{name:*^9}"       →  "***Bob***"         (center, fill with *)
f"{x:-<6}"          →  "42----"            (left-align, fill with -)

# Without the align symbol, fill is ignored
f"{x:05}"           →  "00042"             (special case: 0 before width means zero-pad)
Rule · fill character requires an align symbol after it. 0>5 works; just 05 works only for numbers (special case).

Numbers — precision & type codes

y = 3.14159
big = 1234567

# Float precision (number of decimal places after the dot)
f"{y:.2f}"          →  "3.14"
f"{y:.0f}"          →  "3"                 (rounds — banker's rounding)
f"{y:.4f}"          →  "3.1416"

# Width + precision together
f"{y:10.2f}"        →  "      3.14"        (width 10, 2 decimals)
f"{y:<10.2f}"       →  "3.14      "        (width 10, 2 decimals, left-align)

# Thousands separator
f"{big:,}"          →  "1,234,567"          (comma)
f"{big:_}"          →  "1_234_567"          (underscore)

# Type codes
f"{255:d}"          →  "255"                (decimal int)
f"{255:b}"          →  "11111111"           (binary)
f"{255:x}"          →  "ff"                 (hex, lowercase)
f"{255:X}"          →  "FF"                 (hex, uppercase)
f"{255:o}"          →  "377"                (octal)
f"{0.5:%}"          →  "50.000000%"         (percent)
f"{0.5:.0%}"        →  "50%"                (percent with 0 decimals)

Sign control & the self-documenting debug spec

# Sign — force +/- to show
f"{42:+d}"          →  "+42"
f"{-42:+d}"         →  "-42"
f"{42: d}"          →  " 42"               (space for positive, - for negative)

# Self-documenting f-string (Python 3.8+) — great for debugging
x = 42
name = "Alice"
print(f"{x=}")             →  "x=42"
print(f"{name=}")          →  "name='Alice'"
print(f"{x=:5}")           →  "x=   42"     (you can combine = with normal specs)
The = trick · f"{x=}" prints both the variable name AND its value. Best debug trick in Python. Use it liberally.

Real-world patterns

# 1. Table row with mixed types
i, name, price = 3, "Apple", 1.49
f"{i:3d} | {name:<10} | {price:>8.2f}"
→  "  3 | Apple      |     1.49"

# 2. Pretty money
amount = 1234567.891
f"${amount:,.2f}"           →  "$1,234,567.89"

# 3. Banner with fill
title = "REPORT"
f"{title:=^40}"             →  "=================REPORT================="

# 4. Percentage
ratio = 0.847
f"{ratio:.1%}"              →  "84.7%"

# 5. Progress bar
done, total = 7, 10
bar = "█" * done + "─" * (total - done)
f"[{bar}] {done/total:.0%}"  →  "[███████───] 70%"

Lock these

  1. Recipe order: {value:[fill][align][width][.precision][type]} — left to right.
  2. Alignment symbols: < left, > right, ^ center. Strings default left, numbers default right.
  3. Fill character goes before the align symbol. 0>5 = pad left with zeros to width 5.
  4. Float precision: .Nf for N decimal places. Width + precision: 10.2f.
  5. Thousand separators: :, or :_. Percent: :% or :.0%.
  6. Best debug trick · f"{x=}" prints x=42. Use it everywhere.
