String Methods — Cheat Sheet

Block A.3 of Day 3 · the 7 method families, the traps, and what to actually use
Loop Week · Day 3 · Block A.3

1 · Case methods — change the size of letters

Shouting, whispering, title-casing. All return a new string; original untouched.
s = "Hello World"

s.upper()        →  "HELLO WORLD"     (all big — shouting)
s.lower()        →  "hello world"     (all small — quiet)
s.title()        →  "Hello World"     (first letter of each word big)
s.capitalize()   →  "Hello world"     (only the very first letter big)
s.swapcase()     →  "hELLO wORLD"     (flip every letter's case)
Daily-use ranking · .lower(), .upper() → daily. .title() → book/movie titles. .capitalize() → sentence-style. .swapcase() → once in a blue moon.
Most common use · case-insensitive compare: "Apple".lower() == "apple".lower() → True

2 · Split & Join — inverse twins

Split chops a string into a list. Join glues a list back into a string.
# SPLIT — chop at every separator
"apple,banana,cherry".split(",")  →  ['apple', 'banana', 'cherry']
"hi there friend".split()         →  ['hi', 'there', 'friend']   (no-arg)
"a,b,,c".split(",")               →  ['a', 'b', '', 'c']         (keeps empties!)

# JOIN — tape pieces back together with glue between
",".join(["a", "b", "c"])         →  "a,b,c"
" - ".join(["py", "thon"])        →  "py - thon"

# Inverse pair
",".join("a,b,c".split(","))      →  "a,b,c"   (back to start)
Trap · no-arg vs explicit space. " hi world ".split() → ['hi', 'world'] (smart). " hi world ".split(" ") → ['', '', 'hi', '', '', 'world', '', ''] (strict, keeps empties).
Trap · join only takes strings. ",".join([1, 2, 3]) → TypeError. Fix: ",".join(str(x) for x in [1, 2, 3]).

3 · Strip methods — trim the edges

Cuts unwanted characters from the start and end of a string. Never touches the middle.
"   hello   ".strip()       →  "hello"        (whitespace from both ends)
"   hello   ".lstrip()      →  "hello   "     (LEFT only — l = left)
"   hello   ".rstrip()      →  "   hello"     (RIGHT only — r = right)

# Custom chars to trim
"###hello###".strip("#")    →  "hello"
"<>".strip("<>")     →  "title"        (any of < or >)</code></pre>

  <div class="trap"><b>The big trap · strip never touches the middle.</b><br>
  <code>"  hi  world  ".strip()</code> → <code>"hi  world"</code> (middle spaces stay!).<br>
  To remove all spaces, use <code>.replace(" ", "")</code> instead.</div>
</section>

<!-- 4. FIND / REPLACE / COUNT -->
<section class="card" id="findreplace" style="--accent: var(--findreplace);">
  <h2>4 · Find · Replace · Count — searching and swapping</h2>
  <div class="one-liner">Locate, swap, tally substrings. <code>find</code> forgives misses; <code>index</code> doesn't.</div>

<pre><code># FIND vs INDEX — pick by what you want on a miss
"hello world".find("world")   →  6             (position)
"hello world".find("zebra")   →  -1            (sentinel — no crash)

"hello world".index("world")  →  6             (same when found)
"hello world".index("zebra")  →  ValueError    (CRASHES on miss)

# REPLACE — swap every occurrence (or limit with count arg)
"a b c d".replace(" ", "_")         →  "a_b_c_d"
"a b c d".replace(" ", "_", 2)      →  "a_b_c d"     (only first 2)

# COUNT — non-overlapping occurrences
"ababab".count("ab")          →  3       (not 5 — no overlap)
"aaaa".count("aa")            →  2       (not 3 — no overlap)</code></pre>

  <div class="tip"><b>Pick rule</b> · use <code>.find()</code> if you'll check the result yourself (<code>if pos == -1: ...</code>). Use <code>.index()</code> if you're sure it's there and want a crash if wrong.</div>
  <div class="trap"><b>Count trap</b> · matches don't overlap. After a match, count jumps past it.</div>
</section>

<!-- 5. PREDICATES -->
<section class="card" id="predicate" style="--accent: var(--predicate);">
  <h2>5 · Predicate methods — yes/no questions</h2>
  <div class="one-liner">All return True or False. Check the whole string — one bad char ruins it.</div>

<pre><code># Character-type checks
"12345".isdigit()       →  True       (every char is 0–9)
"abc".isalpha()         →  True       (every char is a letter)
"abc123".isalnum()      →  True       (letters and/or digits)
"abc 123".isalnum()     →  False      (space breaks it!)
"   ".isspace()         →  True       (every char is whitespace)

# Prefix / suffix checks
"hello.py".endswith(".py")            →  True
"hello.py".startswith("hi")           →  False
"hello.py".endswith((".py", ".js"))   →  True   (tuple = "any of these")</code></pre>

  <div class="trap"><b>Empty string is always False</b> · <code>"".isalpha()</code>, <code>"".isdigit()</code>, <code>"".isspace()</code> all → <code>False</code>. They need at least one char to say "yes".</div>
  <div class="tip"><b>Lock</b> · the rule is "every char satisfies it." One bad char = False.</div>
</section>

<!-- 6. PADDING -->
<section class="card" id="padding" style="--accent: var(--padding);">
  <h2>6 · Padding methods — fit a fixed width</h2>
  <div class="one-liner">Pad with characters to reach a target width. Strings only grow — they never shrink.</div>

<pre><code>"42".zfill(5)               →  "00042"        (left-pad with 0s)
"hi".center(10)             →  "    hi    "   (default fill = space)
"hi".center(10, "*")        →  "****hi****"   (custom fill)
"hi".ljust(8)               →  "hi      "     (push left, pad right)
"hi".ljust(8, "-")          →  "hi------"
"42".rjust(5)               →  "   42"        (push right, pad left)
"42".rjust(5, "0")          →  "00042"        (same as zfill)

# Already wide enough? No change.
"hello".rjust(3)            →  "hello"</code></pre>

  <div class="tip"><b>Mnemonic</b> · <code>l</code>just = pushes <b>l</b>eft. <code>r</code>just = pushes <b>r</b>ight. The <i>name</i> says where the content goes, not where the padding goes.</div>
</section>

<!-- 7. F-STRING FORMAT SPECS -->
<section class="card" id="fstring" style="--accent: var(--fstring);">
  <h2>7 · f-string format specs — the mini-language</h2>
  <div class="one-liner">Inside <code>{}</code>, after a colon, you have a tiny formatting language: <code>{value:[fill][align][width][.precision][type]}</code></div>

  <div class="grid-3">
    <div><b><</b> left-align</div>
    <div><b>></b> right-align</div>
    <div><b>^</b> center</div>
  </div>

<pre><code>name = "Alice"
x = 42
y = 3.14159

# Width + alignment
f"{name:10}"          →  "Alice     "     (width 10, default left for strings)
f"{name:>10}"          →  "     Alice"     (width 10, right-aligned)
f"{name:^10}"          →  "  Alice   "     (width 10, centered)

# Fill character (goes BEFORE the align symbol)
f"{x:0>5}"            →  "00042"          (pad left with 0s)
f"{name:*^9}"         →  "**Alice**"      (center, fill with *)

# Numbers — precision for floats, type code for ints
f"{y:.2f}"            →  "3.14"           (2 decimal places)
f"{y:.0f}"            →  "3"              (no decimals, rounds)
f"{y:10.2f}"          →  "      3.14"     (width 10, 2 decimals)
f"{x:5d}"             →  "   42"          (int in width 5)

# Putting it together — a table row
i, name, price = 3, "Apple", 1.49
f"{i:3d} | {name:<10} | {price:>8.2f}"
→  "  3 | Apple      |     1.49"</code></pre>

  <div class="tip"><b>Type codes</b> · <code>d</code> = int, <code>f</code> = float, <code>s</code> = string (default), <code>b</code> = binary, <code>x</code> = hex.</div>
  <div class="tip"><b>Reading a spec</b> · go left to right: fill, then align, then width, then <code>.</code> + precision, then type. Most specs only use 2-3 of these.</div>
</section>

<section class="summary">
  <h2>If you only remember 5 things</h2>
  <ol>
    <li><b>All these methods return new strings.</b> The original never changes. A.1's pen-on-paper lock still holds.</li>
    <li><b>Strip touches only the edges</b>, never the middle. For interior spaces, use <code>.replace(" ", "")</code>.</li>
    <li><b>.find() returns -1; .index() crashes.</b> Pick by whether you want a crash on miss.</li>
    <li><b>.count() does NOT overlap.</b> <code>"aaaa".count("aa")</code> is 2, not 3.</li>
    <li><b>Predicates check the whole string.</b> One bad char = False. Empty string = False for all of them.</li>
  </ol>
</section>
