2 · Split & Join

Inverse twins. Split chops a string into a list. Join glues a list back into a string.
Loop Week · Day 3 · Block A.3 · Subtopic 2

Split — chop at every separator

"apple,banana,cherry".split(",")    →  ['apple', 'banana', 'cherry']
"a-b-c-d".split("-")                →  ['a', 'b', 'c', 'd']

# Limit how many splits with maxsplit
"a,b,c,d".split(",", 2)             →  ['a', 'b', 'c,d']   (only 2 cuts → 3 pieces)

# Multi-character separator works too
"one--two--three".split("--")       →  ['one', 'two', 'three']

The no-arg split — special and smart

"  hi   there   friend  ".split()
→  ['hi', 'there', 'friend']        # smart — collapses any whitespace

"  hi   there   friend  ".split(" ")
→  ['', '', 'hi', '', '', 'there', '', '', 'friend', '', '']
                                    # strict — splits on EXACTLY one space
Rule · .split() with no argument treats spaces, tabs, newlines as one big "whitespace" thing and skips empty pieces. Use this 90% of the time.

Join — tape the pieces back together

# You call .join() ON the glue string. The pieces go inside ().
",".join(["a", "b", "c"])           →  "a,b,c"
" - ".join(["py", "thon"])          →  "py - thon"
"".join(["h", "e", "l", "l", "o"])  →  "hello"          # empty glue
"\n".join(["line 1", "line 2"])     →  "line 1\nline 2"  # newline glue

# Inverse pair — split then join puts it back
",".join("a,b,c".split(","))        →  "a,b,c"
Trap · join only takes strings. ",".join([1, 2, 3]) → TypeError.
Fix: ",".join(str(x) for x in [1, 2, 3]) → "1,2,3".

Real-world patterns

# 1. Reverse words in a sentence (split → reverse → join)
sentence = "the quick brown fox"
" ".join(reversed(sentence.split()))   →  "fox brown quick the"

# 2. Parse a CSV row
row = "Alice,30,engineer"
name, age, role = row.split(",")
                    # → "Alice", "30", "engineer"

# 3. Build a file path
parts = ["users", "alice", "documents", "file.txt"]
"/".join(parts)                       →  "users/alice/documents/file.txt"

# 4. Clean up messy spaces (split + rejoin)
" ".join("  hi   there  ".split())    →  "hi there"

Lock these

  1. Split chops, join glues — they're inverses.
  2. You call .join() ON the glue, with the list inside ().
  3. No-arg .split() is smart with whitespace. .split(" ") is strict.
  4. Join only takes strings. Convert ints first with str().
  5. The four real-world patterns above (reverse words, parse CSV, build paths, clean spaces) cover most use cases.
