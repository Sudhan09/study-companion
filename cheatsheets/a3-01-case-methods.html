1 · Case methods

Change the size of letters. All return new strings; the original is untouched.
Loop Week · Day 3 · Block A.3 · Subtopic 1

The five methods

s = "Hello World"

s.upper()        →  "HELLO WORLD"      (all big — shouting)
s.lower()        →  "hello world"      (all small — quiet)
s.title()        →  "Hello World"      (first letter of each word big)
s.capitalize()   →  "Hello world"      (only the very first letter big)
s.swapcase()     →  "hELLO wORLD"      (flip every letter's case)

# Original unchanged
s                →  "Hello World"

The daily-use pattern — case-insensitive compare

# Don't do this — case matters in ==
"Apple" == "apple"               →  False

# Normalize both sides with .lower()
"Apple".lower() == "apple".lower()   →  True

# Real-world use: input matching, search, sorting
user_input = "YES"
if user_input.lower() == "yes":
    print("confirmed")
Lock · whenever comparing strings where case shouldn't matter, .lower() both sides. Most common case method in real code.

Edge cases & surprises

"".upper()                  →  ""              (empty stays empty)
"123".upper()               →  "123"           (non-letters unaffected)
"HELLO".title()             →  "Hello"         (lowercases the rest!)
"it's me".title()           →  "It'S Me"       (apostrophe triggers re-cap)
"hello world".capitalize()  →  "Hello world"   (lowercases EVERYTHING after pos 0)
Trap · .title() is dumb · it capitalizes after every non-letter, including apostrophes. "it's" → "It'S". For real title-casing, write your own.
Trap · .capitalize() also lowercases the tail · "HELLO WORLD".capitalize() → "Hello world", not "Hello WORLD".

Lock these

  1. All five return new strings. The original is unchanged.
  2. .lower() and .upper() are 95% of real usage. The other three are situational.
  3. Case-insensitive compare: .lower() both sides, then ==.
  4. .title() is dumb about apostrophes — don't trust it on real prose.
  5. .capitalize() lowercases everything after the first character.
